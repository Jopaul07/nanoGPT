# nanoGPT — Decoder-Only Transformer

> **Chapter 7 of Andrej Karpathy's *Neural Networks: Zero to Hero* curriculum.** Builds, from scratch in PyTorch, a character-level decoder-only Transformer language model (GPT) trained on the tinyshakespeare dataset. The pedagogical notebook (`gpt-dev.ipynb`) walks from a bigram baseline through the self-attention mechanism to a full multi-layer, multi-head GPT; the companion `gpt.py` is the productionized single-file script version of the notebook's final cell.

## Core Concepts Demonstrated

1. **Character-level tokenization** — `stoi`/`itos`, 65-token vocab.
2. **Autoregressive framing** — a chunk of `block_size+1` tokens yields `block_size` nested (context → target) examples.
3. **Bigram baseline** — a single `nn.Embedding(vocab_size, vocab_size)` table; chance loss `-ln(1/65) ≈ 4.87`.
4. **The mathematical trick in self-attention** — lower-triangular weighted aggregation via matrix multiplication; cumulative averages via `tril / row_sums` and `zeros + masked_fill(-inf) + softmax`.
5. **Q/K/V self-attention** — `nn.Linear` projections, `wei = q @ k.transpose(-2,-1)`, causal masking, softmax, `out = wei @ v`.
6. **Scaled attention** — `wei * head_size**-0.5` to control softmax variance.
7. **`LayerNorm1d` from scratch** — gamma/beta, mean/var over the feature dim.
8. **Multi-head attention** — parallel `Head`s, concat, project back.
9. **Feed-forward block** — `Linear → ReLU → Linear → Dropout` (the 4× MLP).
10. **Pre-norm residual blocks** — `x = x + sa(ln1(x))`, `x = x + ffwd(ln2(x))`.
11. **Token + learned positional embeddings**.
12. **Context cropping in `generate`** — crop to the last `block_size` tokens to never exceed positional embedding range.
13. **`AdamW` optimizer** — replaces the manual SGD from micrograd/makemore.

## `gpt.py` — The Productionized Script

`gpt.py` (244 LOC) is the **productionized / cleaned-up script version** of the notebook's final cell. It corresponds to Karpathy's repo reference implementation and:

- Contains the same classes (`Head`, `MultiHeadAttention`, `FeedForward`, `Block`, `BigramLanguageModel`) and the same `get_batch` / `estimate_loss` / training-loop structure as the notebook's final cell.
- Hyperparameters (currently tuned down for quick runs): `batch_size=16`, `block_size=32`, `max_iters=5000`, `eval_interval=500`, `lr=1e-3`, `n_embd=64`, `n_head=4`, `n_layer=4`, `dropout=0`.
- Adds `device = 'cuda' if torch.cuda.is_available() else 'cpu'` and `.to(device)` on data/model.
- The `_init_weights` method is commented out (the "better init, not covered in the original GPT video" note).
- After training, generates 500 chars to stdout and 10K chars to `more.txt`.
- Effectively the notebook's final cell packaged as the canonical, runnable reference.

## How to Run

### Run the notebook
```bash
cd nanoGPT
jupyter notebook gpt-dev.ipynb
# or headless:
jupyter nbconvert --to notebook --execute gpt-dev.ipynb --output gpt-dev.ipynb
```

### Run the script
```bash
cd nanoGPT
python gpt.py
```
This trains the GPT for `max_iters` (5000) steps and writes 10K generated chars to `more.txt`.

### Expected runtime
- Notebook (bigram section): ~2 minutes.
- Notebook (full GPT, 5000 iters): ~5 minutes on GPU, ~20 minutes on CPU.
- `gpt.py` (5000 iters): same as notebook's full GPT cell.

## Status & Caveats

- **Everything is complete and working.** Both the notebook and `gpt.py` have been executed with stored outputs.
- `dropout = 0.0` is used (effectively disabled) — typical for the lecture version to keep things stable.
- The `_init_weights` method in `gpt.py` is commented out (Karpathy's "better init, not covered in the original GPT video" note).
- Loss curves are not plotted (no matplotlib); the loss history is only present as printed text.
- The model class is named `BigramLanguageModel` even in the final GPT version — an intentional teaching artifact from the lecture.
- `gpt.py`'s hyperparameters are tuned down (`batch_size=16`, `n_embd=64`, `n_layer=4`) for quick runs. To match the lecture's full-scale results, scale up: `batch_size=64`, `block_size=256`, `n_embd=384`, `n_head=6`, `n_layer=6`, `dropout=0.2`, `max_iters=5000`, `lr=3e-4`.
- The codebase is a clean, faithful study implementation — not extended beyond Karpathy's original yet.
